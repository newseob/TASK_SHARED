# TASK_SHARED 웹앱 작업 노트

## 1. 개요

- 이 프로젝트는 `React + Vite + TypeScript + Tailwind CSS` 기반의 단일 페이지 웹앱이다.
- 메인 목적은 공유 할 일, 루틴, 메모, 링크, 금전 관리 등을 한 화면에서 빠르게 다루는 것이다.
- 데이터 저장은 `Firestore`와 `localStorage`를 혼합 사용한다.
- 탭 기반 구조이며 하단 탭바로 화면을 전환한다.

## 2. 실행/기술 스택

- 실행:
  - 개발 서버: `npm run dev`
  - 빌드: `npm run build`
  - 미리보기: `npm run preview`
- 주요 라이브러리:
  - UI: `React 19`, `Tailwind CSS`
  - 빌드: `Vite`
  - 정렬/드래그: `@dnd-kit`
  - 에디터: `Tiptap`
  - 백엔드 저장소: `Firebase Firestore`

## 3. 전체 화면 구조

- 로그인 화면
  - 앱 진입 시 간단한 비밀번호 입력 화면이 먼저 나온다.
  - 인증 상태는 `localStorage.isAuthenticated`로 유지된다.
- 메인 앱
  - 하단 탭: `할일`, `루틴`, `경인`, `유섭`
  - 다크모드 토글 지원
  - 전체 레이아웃은 단일 브라우저 창 안에서 렌더링되는 웹앱 구조다.

## 4. 탭별 기능 정리

### 4.1 할일 탭

할일 탭은 여러 박스로 구성된 대시보드형 화면이다.

#### 1) TodoBoxSection

- 여러 개의 할 일 박스를 생성할 수 있다.
- 박스별 제목 수정 가능
- 박스 접기/펼치기 가능
- 박스 순서 드래그 정렬 가능
- 박스 삭제 가능
- 각 박스 안의 아이템 드래그 정렬 가능
- 기본 박스와 쇼핑형 박스 두 가지 모드가 있다.
- 쇼핑형 박스는 `이름 / 수량 / 단위` 구조를 가진다.
- 아이템 선택 상태를 토글할 수 있다.
- Firestore 기반 undo 흐름이 연결되어 있어 전역 `Ctrl/Cmd + Z` 대상이 된다.

#### 2) TodayRoutine

- 루틴 데이터 중 `cycle = 1` 인 항목을 일일 루틴으로 보여준다.
- 06:00 기준 날짜 계산 로직을 사용한다.
- 체크 버튼으로 오늘 수행 여부를 토글한다.
- 일일 루틴 목록은 드래그 정렬 가능하다.
- Firestore의 `routineItems/config.items` 데이터를 사용한다.

#### 3) CycleRoutine

- 루틴 데이터 중 `cycle >= 2` 이고 기한이 도래한 항목만 보여준다.
- `remaining` 값을 계산해 `D+n` 형태로 표시한다.
- 마지막 체크일, 마지막 교체일 수정 기능이 있다.
- 기한이 지난 항목을 우선적으로 관리하는 용도다.

#### 4) DateReference

- 루틴 데이터 중 `cycle = 0` 인 참고용 항목을 보여준다.
- 사실상 날짜 참조 박스 역할이다.
- 마지막 체크일/교체일 확인 및 일부 날짜 수정 흐름이 들어 있다.
- 시각적으로는 보조 정보 영역에 가깝다.

#### 5) DietBox

- 식단/레시피 메모용 박스다.
- 로컬 저장(`localStorage`) 기반이다.
- 검색 기능 지원
- 리스트 반응형 배치:
  - 기본 1열
  - `xs(510px)` 이상 2열
  - `md(768px)` 이상 다시 1열
- 리스트는 고정 뷰포트 높이 안에서 스크롤된다.
- 고정(pin) 기능 지원
- 리스트 항목마다 빠른 삭제 버튼 지원
- 새 레시피 생성 가능
- 제목/내용 편집 및 저장 가능
- 제목과 내용이 모두 있어야 저장된다
- 현재 선택된 레시피는 하단 인라인 편집 영역에서 수정한다

#### 6) MoneyBox

- 예산/지출 관리 박스
- 카테고리별 예산과 사용자별 현재 지출 입력 가능
- 사용자 표시 토글 존재
- 메모 및 누적값 관리
- 총 예산, 총 지출 합계 표시
- 변경 이력 기반 undo 연결
- Firestore 저장 문서는 `moneyData/main`

#### 7) LinkBox

- 링크/북마크 관리 박스
- 링크 추가, 수정, 삭제 가능
- 카테고리별 그룹화
- 그룹 접기/펼치기 가능
- 그룹명 수정 가능
- 링크 개별 드래그 정렬 가능
- 그룹 단위 드래그 정렬 가능
- 투명 스타일 토글 지원
- 변경 이력 기반 undo 연결
- Firestore 저장 문서는 `links/main`

#### 8) 박스 순서 편집

- `todo / routine / diet` 3개 상단 박스 순서를 편집 모드에서 위/아래 이동 가능
- 순서는 `localStorage.todoTabOrder`에 저장된다.

### 4.2 루틴 탭

- 루틴 전체 CRUD 화면
- 테이블 기반 편집 UI
- 필드:
  - 이름
  - 메모
  - 마지막 확인일
  - 마지막 교체일
  - 주기
- 인라인 수정 시 Firestore에 바로 저장
- 새 루틴 행 추가 가능
- 삭제 시 확인창 표시
- 정렬 가능
- JSON 내보내기/불러오기 지원
- 저장 문서는 `routineItems/config`

### 4.3 경인 탭 / 유섭 탭

- 두 탭 모두 Tiptap 기반 개인 메모 에디터다.
- 굵게, 색상, 글자 크기 조절 가능
- Firestore에 에디터 JSON 저장
- 저장 문서:
  - 경인: `memos/kyunginMemo`
  - 유섭: `memos/YuseopMemo`

## 5. 데이터 저장 구조

### Firestore

- `sharedData/main.todoBoxes`
  - 할 일 박스 목록
- `routineItems/config.items`
  - 루틴 원본 데이터
- `moneyData/main`
  - 금전 관리 데이터
- `links/main.links`
  - 링크/북마크 목록
- `memos/kyunginMemo`
  - 경인 메모
- `memos/YuseopMemo`
  - 유섭 메모

### localStorage

- 로그인 상태: `isAuthenticated`
- 다크모드: `theme`
- 각 박스/섹션 펼침 상태
- Todo 상단 박스 순서
- DietBox 레시피 목록 및 선택 상태
- 일부 그룹 접힘 상태 등 UI 상태 전반

정리하면:
- 공유되어야 하는 핵심 데이터는 Firestore
- 개인 UI 상태와 일부 가벼운 데이터는 localStorage
- DietBox 레시피는 현재 Firestore가 아니라 localStorage에만 저장됨

## 6. 공용 동작/유틸

### 글로벌 Undo

- `useGlobalUndoScope` 훅으로 전역 undo 스코프를 관리한다.
- 편집 가능한 input/textarea 안이 아닐 때 `Ctrl/Cmd + Z`를 가로채 가장 최근 스코프의 undo를 실행한다.
- 현재 undo가 연결된 대표 영역:
  - TodoBoxSection
  - MoneyBox
  - LinkBox
  - `useFirestoreHistory`를 사용하는 영역들

### Firestore 이력 훅

- `useFirestoreHistory`는 Firestore 동기화 + 로컬 히스토리 + undo를 함께 제공한다.
- 실시간 구독(`onSnapshot`)과 로컬 히스토리 관리가 동시에 들어가 있다.

## 7. 최근 작업 노트

최근 식단 박스 쪽에서 반영된 작업:

- 리스트 반응형 열 수 기준을 Todo 박스와 동일하게 맞춤
- 레시피 목록은 최대 높이 안에서 내부 스크롤되도록 조정
- 선택 시 배경 강조 제거
- 리스트 제목 글자 크기 상향
- 제목/내용이 비어 있으면 저장 비활성화
- `+ 새 레시피` 버튼을 배경 없는 텍스트 버튼으로 변경
- 저장/삭제 버튼을 회색 계열 텍스트 중심으로 정리
- 각 레시피 아이템에 `고정` 옆 `삭제` 버튼 추가
- 레시피 개별 삭제 시 확인창 표시

## 8. 현재 구조상 주의할 점

- 로그인은 실제 보안 인증이 아니라 클라이언트 측 비밀번호 비교다.
  - 즉 보안 장벽이라기보다 간단한 접근 제한 수준이다.
- Firebase 설정이 프론트 코드에 포함되어 있다.
- Firestore와 localStorage가 섞여 있어, 브라우저를 바꾸면 일부 상태가 달라질 수 있다.
- DietBox 데이터는 아직 공유 저장소가 아니라 브라우저 로컬에만 저장된다.
- 코드 일부에 한글 문자열/주석 인코딩 깨짐이 보인다.
  - 기능 자체보다 유지보수성과 가독성에 영향을 줄 수 있다.
- `src/saveDataToFirestore.ts`는 현재 구조상 레거시 성격이 강해 보이며, 사용 여부를 재점검할 필요가 있다.
